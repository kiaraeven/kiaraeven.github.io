Introduction to function types in cpp

Member Functions

Functions that belong to a class and operate on its data members. It has access to all class members, including private data.

Key Points:

  • Can access all members (public, private, protected).
  • Most member functions are called on an instance using objectName.functionName(). Static functions are called on the class (ClassName::Function()).
  • Can be defined inside (in .h files directly) or outside (declare the functions in .h files and define them outside using the ClassName::FunctionName syntax in .c files) the class.

Static Member Functions

Static functions belong to a class but do not operate on instance-specific data. They are useful for utility functions that do not require an instance of the class.

Key Points:

  • Declared with static inside the class.
  • Can only access static members (The only condition for a static function to access a member (function and data member) is that the member must also be static. It has nothing to do with whether the member is public, private, or protected).
  • Called using ClassName::FunctionName(), but can also be called via objects.

Use Cases:

  • Utility functions related to the class but not specific to any object.

  • Keeping track of object counts, configuration settings, singleton patterns, etc.

Inline Functions

An inline function is a function that the compiler replaces directly at the point of the function call to avoid function call overhead.

When to use inline functions?

  • When the function is small (e.g., one or two lines of code).
  • When function call overhead needs to be minimized.
  • Avoid inlining large functions as it can increase binary size.

Key Points:

  • Suggested using the inline keyword, but modern compilers decide whether to actually inline it.
  • Works best for short, frequently called functions.
  • Inlining large functions can increase binary size (code bloat).

Const Member Functions

A member function that does not modify the state of an object. It guarantees that no member variables (except mutable ones) will be changed inside the function.

Examples

  1. Marking getter functions as const is a best practice since they should not modify the object.

Key Takeaways

  • A const member function ensures the object’s data remains unchanged.
  • Const objects can only call const functions (cause compilation error). Non-const objects can call both const and non-const functions.
  • Use mutable to mark the variables that must change inside const functions for exceptions (e.g., caching, logging).
  • Always mark getter functions as const for safety.

Virtual Functions

A virtual function is a member function that can be overridden in a derived class, allowing dynamic (runtime) polymorphism.

Key Points:

  • Virtual functions enable runtime polymorphism.

  • Use override keyword after the parameter list to avoid accidental function signature mismatches.

  • Always use a virtual destructor in a base class if you expect it to be inherited.

    • If a base class destructor is not virtual, deleting a derived class object through a base class pointer only calls the base class destructor. If Derived allocated dynamic memory, it would not be freed, leading to a memory leak.
    • Incorrect example:
    • Why Does This Happen?
      • When using virtual ~Base(), the destructor lookup is done at runtime (dynamic dispatch), ensuring the most derived destructor is called first.
      • Memory Deallocation Flow (destruction order (derived -> base))
        • The delete ptr; statement triggers Base’s destructor.
        • Since it’s virtual, it first calls Derived’s destructor.
        • After Derived’s destructor completes, Base’s destructor runs.
        • All allocated resources are properly released.

Lambda Functions

A lambda function is a short, unnamed function used for quick operations, especially in algorithms and functional programming.

Lambda Function Syntax

[capture_list] specifies which variables from the surrounding scope should be captured.

(parameters) defines the function’s parameters.

-> return_type (optional) specifies the return type.

{ function_body } contains the function logic.

Capture list and parameters

  • Capture List ([]): Used for accessing external variables from the surrounding scope.
    • Key Takeaways
      • Capture by Value ([x]): Read-only copy.
      • Capture by Reference ([&x]): Modifications affect the original variable.
      • Capture Everything by Value ([=]): All local variables (except static or global variables, but lambda functions can access these variables normally already) are captured by value. This makes the lambda safe for multi-threading but prevents modifications unless mutable is used.
      • Capture Everything by Reference ([&]): All local variables are captured by reference, allowing modifications inside the lambda to affect the original variables.
      • Mis Capture [=, &y]: Captures all by value, except y by reference.
  • Parameter List (()): Used for passing arguments like a normal function.
  • Captured values are fixed at the time the lambda is created, while parameters change per call.

Use Cases

  • Callback functions in algorithms (std::for_each, std::sort).

  • Threading (std::thread).

    • Example

  • Functional programming paradigms (e.g., std::transform to apply a given operation (usually a lambda or function) to each element in a range (e.g., array, vector, list) and stores the results in another range.).
