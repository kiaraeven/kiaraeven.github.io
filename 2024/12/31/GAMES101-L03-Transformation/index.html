GAMES 101 L03-Transformation

Common Transformation Matrices

  1. Scale Matrix (non-uniform)
    Essence: x′(xprime)=sxx,y′=syyx'(x_{prime}) = s_xx, y' = s_yyx′(xprime​)=sx​x,y′=sy​y => (represent in matrices)
    “non-uniform” scaling means that the scaling factors applied to the x-axis (sx) and y-axis (sy) are not equal.
  2. Reflection Matrix
    x′(xprime)=−x,y′=yx'(x_{prime}) = -x, y' = yx′(xprime​)=−x,y′=y
  3. Shear Matrix
  4. Rotation Matrix
    Without spacific instructions, default center is origin and default rotating direction is counter clock-wise.

    For rotation matrix MMM, if we want an inverse rotation (from the same center rotating the same degree in a different direction), the new rotation matrix should be M−1M^{-1}M−1 (by the definition of inverse transforms). In addition, if we apply −θ-\theta−θ to the rotation matrix of θ\thetaθ, we will find that the new rotation matrix can also be reprensented as MTM^TMT. That is to say, for a rotation matrix MMM, M−1=MTM^{-1} = M^TM−1=MT.

Summary
Above transformations can be represented in linear combinations. Linear transformations = Matrices (in the same dim as the transformed object).

Homogeneous Coordinates (齐次坐标)

Why introduce homogeneous coordinates

Considering translations, which can be represented as x′(xprime)=x+tx,y′=y+tyx'(x_{prime}) = x + t_x, y' = y + t_yx′(xprime​)=x+tx​,y′=y+ty​, they cannot be represented simply using a 2d matrix. However, we don’t want to treat translation as a special case, so we introduce homogeneous coordinates, to represent adding a constant.

As shwon in the figure above, why are the representations of points and vectors different? This is because vectors are translation invariant. Adding a 0 instead of 1 at the end is to protext vectors from changing caused by translations. In this case, if we use a translation matrix to multiply the vector, the vector will remain unchanged.

Definitions


In the context of homogeneous coordinates, Point + Point representation the midpoint between the two points.

Affine Transformations (仿射变换)

Use affine transformations to combine linear transformations and translations. Note that when using affine transformations the last row is always [0,0,1][0, 0, 1][0,0,1] (only applied to affine transformations, for projection transformations the last row has other meanings) and the last column is always [tx,ty,1]T[t_x, t_y, 1]^T[tx​,ty​,1]T. And the top left four elements represents the linear transformations.

Inverse Transformations (逆变换)

M−1M^{-1}M−1 is the inverse of transform MMM in both a matrix and geometric sense. Imagin we use matrix MMM to transform a point, to revert it to the original state, we apply M−1M^{-1}M−1 to the point.

Composing Transformations

Compose simple transforms to form a complex transform. Transform ordering matters. Note that matrices are applied from right to left. Matrices don’t have the communative law but have the associative law, so we can multiplie all the transform matrices to get one composite matrix.

3D Transform

Similarly, in 3D Transforms, vectors are translation invariant.

We use a 4×44 \times 44×4 matrix to represent 3d affine transforms. For 3d affine transforms, the last row is always [0,0,0,1][0, 0, 0, 1][0,0,0,1], the last row is always [tx,ty,tz,1]T[t_x, t_y, t_z, 1]^T[tx​,ty​,tz​,1]T, while the left top 3×33 \times 33×3 matrix represents the 3D linear transforms. Note that for an affine matrix (both 2d and 3d), if we decompose the matrix, it should be applying linear transform first, and then the translation.

3D rotations are similar to 2D rotations, except for the rotation about y axis. This is because y⃗=z⃗×x⃗\vec{y} = \vec{z} \times \vec{x}y​=z×x.

For complex rotations, we can decompose them to rations about each axis.

A more general method for 3d rotation is Rodrigues’ ratation formula (axis n passes origin by default).
